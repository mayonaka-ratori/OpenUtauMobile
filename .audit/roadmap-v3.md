# OpenUtau Mobile フォーク改良 — 優先度別ロードマップ（v3 統合版）

## 前提：競合基準線と勝ち筋

モバイルでの直接競合基準線は引き続き **VoiSona iOS**。  
評価基準は「AIの有無」より先に **“インストールから30秒で音が鳴るか”** とする。[12]

ただし OpenUtau Mobile の勝ち筋は、VoiSonaの単純コピーではない。

- **UTAUの既存音源資産をモバイルで使える**
- **無料・オープンソース・Android対応**
- **PC版OpenUtau / USTX / DiffSinger と往復できる**
- **手調声とAI補助の中間地帯を取りにいける**

---

## 設計原則（v3で追加）

1. **壊れないより先に、壊れても戻れる**  
   → オートセーブ / 復元 / セーフモード

2. **観測できない不具合は直せない**  
   → クラッシュ収集 / パフォーマンス計測 / ログ出力

3. **互換性問題は実行時ではなくインポート時に潰す**  
   → 音源診断 / 文字コード / OTO異常検知

4. **モバイルでは“パス”を信用しない**  
   → 資産レジストリ / URI永続化 / project package

5. **AIは全自動の前に補助から入る**  
   → ピッチテンプレ / スタイルプリセット → その先に full AI

# 優先度変更まとめ（v2 → v3）

## 昇格した項目
| 項目 | 変更理由 |
|:---|:---|
| クラッシュ・性能テレメトリ | メモリ改善や再現不能バグ修正の前提だから |
| 音源インポート時の互換性診断 | cutoff防御だけでは後手。サポート負荷を大きく減らせる |
| 資産レジストリ / パス抽象化 | クラウド同期やSAF対応の前提。後回しにすると負債化する |
| レンダリングジョブ管理 | プレビュー体験・電池・CPU・UI応答性の中核だから |
| 適応レイアウト / タブレット基盤 | テーマより先に、モバイルで破綻しないUIが必要 |
| ピッチアンカー編集 | 自由曲線よりモバイル適性が高いから |

## 優先度を下げた項目
| 項目 | 理由 |
|:---|:---|
| UIテーマ | 適応レイアウト未完成の段階では後回しでよい |
| SideStore/AltStore対応 | 有効だが玄人向け。勝ち筋の中核ではない |
| full AI自動ピッチの即時実装 | まずはテンプレ・補助層の方が工数対効果が高い |
| ENUNU | 依然として高コスト・低リターン |

## 統合・拡張した項目
| 旧項目 | 新しい扱い |
|:---|:---|
| 0-1 オートセーブ | セッション復元・セーフモード込みに拡張 |
| 0-3 cutoff防御 | import validator と詳細UIへ接続 |
| 1-8 BGM読み込み安定化 | 圧縮音源対応まで拡張 |
| 1-10 デモ音源同梱 | チュートリアル・サンプルプロジェクトと統合 |
| X-1 チュートリアル | 1-1に統合して初回体験の中核へ |
| 2-2 Luaフォネマイザ | App Store安全性を考慮し DSL fallback を追加 |
| 3-4 AIピッチ生成 | 2-10 の非AI補助層を先に置いた段階設計に変更 |
